## Cheat sheet voix de marque

### Ton
Technical, direct, and confident — like a senior developer recommending a tool to a peer. No fluff, no hype. Show, don't tell.

### DO
- Use code snippets and terminal commands as proof
- Be specific with numbers ("6 tools", "3 dependencies", "5 kB")
- Use the audience's own language (from market research verbatims)
- Lead with the "why" (agent needs real-world tools) then the "how" (MCP)
- Keep sentences short and scannable

### DON'T
- No marketing buzzwords ("revolutionize", "game-changing", "supercharge")
- No fake urgency ("limited time", "act now")
- No vague claims without proof ("best", "fastest", "most powerful")
- No enterprise jargon ("enterprise-grade", "scalable solution", "leverage")
- No dashboard screenshots — show code and terminal output
- No emojis

|-----------|--------|----------------------|

### Mots à éviter

| Mot/expression | Pourquoi l'éviter | Alternative |
|---------------|-------------------|-------------|
| "best QR code generator" | Saturé, tous les concurrents le disent | "QR codes built for AI agents" |
| "enterprise-grade" | Pas crédible pour un nouveau produit | "production-ready" |
| "revolutionize" / "game-changing" | Buzzword, devs allergiques | Montrer un code snippet |
| "free forever" | Associé aux arnaques QR (bait and switch) | "Free tier. No tricks." |
| "branded QR codes" | Commodity, pas le différenciateur | "dynamic QR codes" |
| "scalable" | Vague, overused | Chiffrer : "handles X requests/sec" |
